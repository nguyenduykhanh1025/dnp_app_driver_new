HTML5 document type

Some HTML elements and CSS attributes used by Bootstrap need to set the page to HTML5 document type. Each page in your project should be set according to the following format.

Mobile first

In Bootstrap 2, we added mobile-friendly styles to some key parts of the framework. In Bootstrap 3, we rewrote the entire framework to make it mobile friendly from the beginning. This time, it is not simply adding some optional styles for mobile devices, but directly integrated into the core of the framework. In other words, Bootstrap is mobile-first. Instead of adding an extra file, the style for mobile devices is integrated into every corner of the frame.

Composition and link

Bootstrap typography and link styles set basic global styles. They are: Set the background-color for the body element: #fff; Use the @font-family-base, @font-size-base and @line-height-base a variables as the basic parameters of typography. Set the basic color for all links @ link-color, and only add underscores when the link is in the :hover state. These styles can be found in the scaffolding.less file.

Normalize.css

To enhance the consistency of performance across browsers, we used Normalize.css, a CSS reset style library maintained by Nicolas Gallagher and Jonathan Neal.

Layout container

Bootstrap needs to wrap a .container container for the page content and grid system. We provide two classes for this purpose. Note that these two container classes cannot be nested with each other due to attributes such as padding.

Grid system

Bootstrap provides a responsive, mobile device-first streaming grid system. As the size of the screen or viewport increases, the system will automatically be divided into a maximum of 12 columns. It contains easy-to-use predefined classes and a powerful mixin for generating more semantic layouts.
